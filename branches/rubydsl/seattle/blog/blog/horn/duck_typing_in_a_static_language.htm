In the previous post I explained how the horn software system inserts an anonyomous base class into the pipeline in order to effectively become the base class which all Dsl scripts will inherit from. I now want to explain how the following syntax was achieved 
dependencies:
    package.homepage = "http://code.google.com/p/scotaltdotnet/"
    package.forum    = "http://groups.google.co.uk/group/horn-development?hl=en"
    package.contrib  = false
You might be wondering what in the world is unusual about this code and you would be correct to query the worth of explaining such trivial code. It seems even more trivial when you examine the horn source code and discover that there is a Package object which is defined below: 
class Package(IQuackFu): 
    [property(PackageInfo)]
    public packageInfo as Dictionary[of string, object]
 
    def QuackGet(name as string, parameters as (object)) as object:
        pass
 
    def QuackInvoke(name as string, args as (object)) as object:
        pass
    
    def QuackSet(name as string, parameters as (object), value) as object:  
        packageInfo.Add(name, value)
    
    def constructor():
        packageInfo = Dictionary[of string, object]()
The first thing of note is that there is not a homepage method or a forum method or indeed a contrib method but yet the horn dsl script compiles and is able to add the values to the horn domain model. You might also note above that the Package object implements the IQuackFu interface which is quite possibly the best named interface the .NET framework. The IQuackFu is boo's answer to method missing method not understood in dynamic languages.
